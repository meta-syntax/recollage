# ADR-002: カード密度は保存せず導出する

- ステータス: 採用
- 日付: 2026-06-11

## コンテキスト

フィードのカードは可変密度（構造化の度合いでサイズ・情報量が変わる）。「構造化は強制ではなくインセンティブ——埋めるとビジュアライズが良くなる」というプロダクト原則を、データ設計でどう支えるか。

## 決定

密度（大・中・小）をカラムとして保存せず、フィールドの充足状況から導出する。

| 密度 | 条件 |
|---|---|
| 大 | visual あり |
| 中 | title と keyPoints（1件以上）あり |
| 小 | 上記未満 |

導出ロジックは `shared/utils/` の1関数に集約し、フィード編成とカードコンポーネントが同じ判定を共有する。

## 帰結

- 「フィールドを埋めるとカードが育つ」が自動で成立する。密度の更新処理・整合性管理が不要
- 書き捨てエントリは物理的に小さく表示されるため、フィードの見た目を大きく損なわない（書き捨て問題の緩和）
- 密度の判定基準を変えたいときは導出関数1箇所の変更で全体に反映される
